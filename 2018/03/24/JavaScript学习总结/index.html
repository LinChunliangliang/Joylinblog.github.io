JavaScript

简单介绍咯

​ 是一个轻量级，基于对象，事件驱动，客户端脚本语言，其严格区分大小写

字符串相关函数

parseInt()在一个字符串中从左向右提取整数，直到遇到第一个非数字旧结束提取

parseFloat()在一个字符串中从左向右提取小数

isNaN()判断一个字符串是不是有纯数字组成，如果是返回true否则则返回false

作用域

​ 作用域指的是，变量可以被引用的范围，不同作用域下声明的变量可引用的范围不同。

​ 在全局作用域声明的变量叫全局变量，在局部作用域下声明的变量称为局部变量

作用域链的原则

  1. 当试图引用一个变量时需要遵循作用域链原则。尝试引用一个变量时，先在当前作用域寻找该变量的声明语句var 变量名,如果有直接使用该变量
  2. 如果没找到，继续向上一级作用域寻找var声明语句。如果有，直接使用
  3. 如果还是没有，继续向上一级作用域寻找var声明语句。如果有，直接使用
  4. 周而复始，如果找到全局作用域还是没有，系统会自动地，隐式创建该变量的声明语句

闭包

​ 闭包主要解决跨函数 引用变量的问题，低级作用域可以引用高级作用域的变量 高级作用域不能引用低级作用域的变量

递归

递归就是函数在内部调用自己

主要用于解决，一个大的事件拆分成小事件，但是大小事件的解决方式都一样，这样的特殊事件就可以用递归解决

延时器和定时器

延时器只是在一段时间后执行某个事件，只执行一次

定时器是没隔一段时间做某事

事件监听

​ 如果用动态绑定 无法给同一个网页元素 多次绑定同一个类型的事件

​ 事件监听允许我们给同一个网页元素 的 同一种事件类型 绑定多个处理程序

IE和w3c在事件监听中存在兼容性问题，以下封装解决

event事件对象

​ event事件对象 在事件触发时 系统会自动产生的一个临时对象

​ event事件对象会保存与当前事件类型密切相关的重要数据

事件冒泡与阻止事件冒泡

​ 用户的一个行为 可能同时触发多个元素的多个事件

​ 阻止事件冒泡

​ W3C内核:

​ event事件对象.stopPropagation()

​ IE内核:

​ window.event.cancelBubble = true
