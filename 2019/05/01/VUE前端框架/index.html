VUE前端框架

使用VUE

首先需要通过<script>标签引入Vue框架脚步

两步：

  1. 定义挂载点<div id="app"></div>
  2. 创建Vue实例
    1. 说明挂载点el:"#app"
    2. Vue相关定义操作
标签 相关说明
el 定义挂载点
data 定义数据
methods 定义方法，函数
template 定义模版
computed 定义计算属性
watch 定义侦听器
filters 私有过滤器
directive 私有指令

##Vue相关指令

vue指令指的是vue提供特殊的html标签属性，有v-前缀

vue指令 相关说明
v-text 在标签中渲染文本数据
v-html 在标签中渲染html文本内容，会解析html标签
v-bind 给属性绑定数据
v-on 事件绑定

<button v-bind:title="msg" v-on:click="demo>按钮</button>"

<button :title="msg" @click="demo>按钮</button>"

msg这个数据需要在data中定义

其中demo()这个函数需要在methods中定义

###vue事件修饰符

v-on:事件类型.事件修饰符 = @事件类型.事件修饰符

事件修饰符 相关说明
.stop 阻止冒泡
.prevent 阻止默认行为
.once 绑定一次性事件

##表单数据双向绑定

针对表单数据，我们有读和写的功能，所以对于表单数据，我们需要进行双向绑定，即其中一边修改了，另外一边也会进行相应修改

账户<input type='text' v-model="username">

username需要在data中定义即可

数据绑定修饰符

事件修饰符 相关说明
.lazy 懒同步，在填写结束时才进行一次数据同步工作
.trim 去空
.number 取得文本框内容时，数据一般都是字符串型，可以把填写的值自动转化为数值

计算属性

data中保存的一般是原始数据,computed中经过处理的数据

计算属性的本质是函数，函数需要对原始数据进行计算 把计算结果当做返回值

调用{{fullname}}

计算属性和函数的不同

  1. 计算属性调用时不需要加括号
  2. 函数调用时需要加括号
  3. 计算属性在第一次被调用后，会把结果缓存起来，留作将来使用（大大提高性能）

侦听器

watch

侦听器可以监控数据的变化 如果数据发生了变化 对应的侦听器会被调用

（监听器的本质是函数 侦听数据发生改变会调用）

v-for指令

查 读取数据 查多条时，数据是批量的

可以遍历一维数组，也可以遍历多维数组

v-for指令与key的注意事项

可能在使用 v-for 时提供 key，除非遍历输出的 DOM 内容非常简单，或者是刻意依赖默认行为以获取性能上的提升

##v-if&v-show

v-if&v-show:控制元素显示和隐藏 建议使用v-show

数据过滤器

定义全局过滤器

以下是一个私有的时间过滤器

调用形式{{数据|过滤器}}

按键修饰符

<input v-on:keyup.13="submit">等于<input v-on:keyup.enter="submit">

自定义封装指令

定义全局指令Vue.directive(指令名，函数)

定义指令时,指令名不要带前缀,使用 在标签属性中调用指令 调用指令时,指令名要带前缀

定义私有指令：在vue实例内部,通过directives配置属性进行定义,只能在该定义指令的实例中使用

钩子函数

钩子函数 钩子函数在特殊的时间点会自动触发的函数

定义：Vue.directive(指令名,钩子函数) bind钩子函数

Vue生命周期钩子函数

1、new Vue实例化
beforeCreate 不能访问data中的数据 不能调用methods中的方法
2、创建vue实例完成
created 能够访问data中的数据 可以调用methods中的方法（能够把数据保存(赋值操作)到data中的最早时机）

​ beforeMount 模板中的内容还是原文
3、首次渲染数据(挂载)
​ mounted 模板中的内容已经是动态数据（能够使用$refs中保存的dom元素的最早时机）

​ beforeUpdate 模板中的内容是旧数据
4、更新数据
​ updated 模板中的内容是新数据
5、销毁

组件

###组件的定义

组件指的是可复用的VUE实例

在Vue中可以通过Vue.component(组件名，组件配置对象)来创建公有组件

父子组件数据通信

prop属性的作用 把组件标签属性 注册为组件数据

父子组件，即父组件的数据可以给子组件使用

Vue-router

###使用步骤

  1. 引入框架和插件
  2. 创建当前页面所需组件
  3. 实例化VueRouter类，编写多条路由规则(routes) path对应component
  4. 把路由规则与vue捆绑起来，关联起来(router)
  5. 设置显示组件 视图区域<router-view>

###router-link标签

路由插件提供了 < router-link >标签 可以制作跳转链接

​ to: 说明跳转路径 不用带#

​ tag: 用什么元素呈现跳转链接 (span div button)

redirect重定向

{path:"/",redirect:"/news"}

即设置访问”/“时让用户转访问”/news”

###组件库

####组件插槽

插槽 是一个通信渠道 组件数据传递通道，可以解决 组件标签中写内容的问题，组件标签中的内容 会被保存在插槽中

可以在组件模板template中，用 <slot>标签 取出插槽中的内容

具名插槽

具名插槽的作用 是可以给数据分别起名字 分别保存到不同的具名插槽中

用template标签包裹数据 并 用slot属性进行命名

用slot标签读取数据 用name属性说明数据名
