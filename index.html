webpack

安装webpack的前提是需要安装npm工具或者cnpm工具

Start

指令 相关说明
cnpm view webpack versions 查看webpack可用版本
cnpm view webpack version 查看webpack已安装版本
cnpm i webpack@版本号 -g 安装webpack
cnpm i webpack-cli -g 安装webpack-cli

webpack打包

webpack打包的原理 把多个代码文件内容抽离出来 全部放在打包结果文件中

webpack4打包：webpack 输入文件路径 -o 输出文件路径 –mode 打包模式

main.js中先载入其他js函数库等

webpack ./main.js -o ./bundle.js —mode development

webpack配置

步骤

  1. 先卸载全局下的webpack:cnpm remove webpack webpack-cli -g

  2. 在项目本地安装webpack和webpack-cli:cnpm i webpack webpack-cli -D

  3. 在项目目录下创建webpack配置文件webpack.config.js

  4. 安装jquery:cnpm i jquery -s,在main.js文件中载入jquery模块（详情见12步）

  5. 在webpack的配置文件中，说明输入和输出文件

  1. 在npm配置文件，添加新脚本，使用项目本地安装的webpack软件

  2. 调用npm的脚本npm run dev即可使用,但是每改变代码一次就要重新调用，所以需要安装webpack-dev-server开发服务，类似于nodemon

  3. 安装webpack-dev-server:cnpm i webpack-dev-server -D

    安装好后需要在npm配置文件中把webpack修改为webpack-dev-server

  1. Web-dev-server打包的产出，放在服务的根目录下，其实时一个临时文件，放在内存里，需要修改index.html的引用路径

    <script src="/bundle.js"></script>

  2. web-dev-server详细参数

  1. 主页虚拟化html-webpack-plugin:cnpm i html-webpack-plugin -D

    在webpack.config.js文件中配置导入插件并且配置

  1. webpack打包css

    • 在项目中添加css样式文件index.css

Ajax学习总结

##简介

​ ajax是一种保障网页数据实时性的技术，意思是数据从服务端产生的，并且不刷新的情况下可以更新数据。

ajax（asynchronous javascript and xml）

使用Ajax五步走

第一步：创建ajax对象

创建ajax对象时，会存在兼容两个内核版本（IE内核，W3C内核）的不同，导致两个创建ajax不一样

IE内核：new ActiveXObject()

W3C内核：new XMLHttpRequest()

由于两个版本的不同，所以我们需要封装一个新的，解决兼容性问题

使用时直接调用

var ajax=crearAjax()

第二步：初始化设置

ajax.open('get','http://locallhost:3000')

这里的请求方式有两种，get和post

不同点：

  1. 数据位置不同
  2. post有特有的请求头，所以使用时需要设置ajax.setRequestHeader("Content-type","application/x-www-form-urlencoded")
  3. post请求数据不会出现在URL中，是通过send方法中，get请求的数据会出现在URL中

发送请求

ajax.send(null)

接收响应

ajax的通信状态码ajax.readyState，状态码总共5种

  1. 0:创建ajax对象，还未初始化调用了creatAjax方法，还未调用open
  2. 1:进行了初始化设置，还未发送请求，调用了open还未调用send
  3. 2:发送了请求，还未收到任何响应，调用了send还未收到任何响应
  4. 3：开始收到响应，响应还未接收完毕
  5. 4:响应接收完毕

HTTP响应状态码ajax.status

  1. 200 成功
  2. 304 延用缓存
  3. 404 页面未找到
  4. 500 服务器内部错误

使用响应数据

响应的数据会存在responseText中

前后端交互注意事项

后端响应必须传入字符串，二进制数据（需要转化），json格式数据，xml数据格式

IE存在缓存问题

解决办法：

  1. 可以在请求地址中加随机数 Math.random() [0,1)
  2. 可以在请求地址中加时间戳/毫秒数 (new Date).getTime()
  3. 通过设置请求头 禁止在浏览器产生缓存
    ajax.setRequestHeader(“If-Modified-Since”,”0”)
    强制让所有的缓存过期 浏览器必须从服务器上读取真实资源

跨域访问

解决的问题是，一个网站（网站A）向另外一个网站（网站B）发送请求时，网站B会拒绝网站A的请求，因为这里发生了跨域请求，当发生跨域请求时，浏览器会报错

JSONP跨域

JSONP跨域技术主要是利用script标签发起的请求不受同源策略限制的这个漏洞

步骤

  1. 搭建好通信渠道，避开同源策略

    在服务器a中设置

    <script src='http://www.b.com:4000'></script>

  2. b服务器负责提供数据

    json的请求一定是符合js‘语法的响应字符串

  3. 实现a自由使用数据

  4. 解决b服务器函数名固定不变的问题

    在访问b服务器的src中，带一个回调函数进行访问

    <script src='http://www.b.com:4000?callback=useData'></script>

CORS跨域

通过服务端设置响应头方式解决跨域需求(Access-Control-Allow-Origin)

B服务器可以设置说明

  1. 允许任意源前来访问

    Access-Control-Allow-Origin:*

  2. 代表允许http://www.a.com前来访问

    Access-Control-Allow-Origin:http://www.a.com

当设置CORS头之前，需要排除之前解决缓存问题设置的if-modified-since的值，避免冲突
